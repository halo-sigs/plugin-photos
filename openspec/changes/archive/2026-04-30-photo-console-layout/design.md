## Context

当前 `PhotoList.vue` 使用左右分栏布局：左侧 `VCard` 包裹的 `GroupList`（固定宽度 384px），右侧照片网格。照片卡片也使用 `VCard`，样式较重。整体空间利用率低，照片展示区域被压缩。

## Goals / Non-Goals

**Goals:**
- 最大化照片展示区域（全宽布局）
- 分组选择器轻量化为顶部横向控件
- 照片卡片更紧凑、更现代
- 所有现有功能（上传、批量操作、搜索、分页、视图切换）在新布局中正常工作

**Non-Goals:**
- 改变后端 API 或数据模型
- 新增功能（标签、EXIF 展示等在 photo-upload-refactor 已实现）
- 修改主题端 `/photos` 页面

## Decisions

**1. 分组选择器放到顶部**
- 选择：横向分组标签列表（类似浏览器标签或文件管理器路径栏）
- 理由：节省左侧空间，用户一眼看到所有分组，点击即可切换
- 分组管理（新建/编辑/删除/排序）通过分组旁边的下拉菜单或按钮触发

**2. 卡片用 TailwindCSS 重写**
- 选择：纯 div + Tailwind 工具类，不用 `VCard`
- 理由：VCard 有 padding、border、shadow 等预设样式，不够灵活；Tailwind 可以精确控制每个像素
- 设计：圆角、hover 时轻微放大和阴影、名称浮层在底部、选中状态用 ring 或角标

**3. 列表视图保持表格形式**
- 选择：视图切换按钮在工具栏中，列表视图使用 `VTable` 或原生 table
- 理由：批量管理时列表视图效率更高，不应删除

**4. 批量操作工具栏与搜索栏整合**
- 选择：顶部固定工具栏行，包含：分组选择器 | 搜索 | 视图切换 | 新增下拉 | 批量操作（选中后显示）
- 理由：所有操作集中在一行，空间利用率高

## Risks / Trade-offs

- **[Risk]** 分组数量很多时顶部横向放不下 → **Mitigation**：分组过多时自动折叠为下拉选择器，或横向滚动
- **[Risk]** 用户习惯了左侧分组列表 → **Mitigation**：新布局更直观，且操作路径更短（少一次视线移动）
